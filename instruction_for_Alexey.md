## 1. Архитектура и доступ

### Где находится раздел
- **URL пользователя:** `app.listoshenkov.ru/private/`
- **URL админки:** `app.listoshenkov.ru/private/admin.html`
- **Карточка «Ближний круг»** в меню навигатора сейчас **скрыта** — пользователи попадают только по прямой ссылке или через бота

### Кто имеет доступ
| Роль | Что видит |
|------|-----------|
| Пользователь без подписки | Лендинг с тарифами |
| Пользователь с активной подпиской | Полный SPA: хаб + все табы |
| Алексей (user_id=1, role=admin) | SPA + кнопка «⚙️ Админка» в сайдбаре |

### Как определяется доступ
1. Пользователь авторизован в основном приложении — токен в `localStorage.session_token`
2. При открытии `/private/` — приложение запрашивает `check_access` через API
3. API проверяет `user_access.has_inner_circle = true` И `inner_circle_expires > now()`
4. Если оба условия — показывается SPA, иначе — лендинг

### Переключатель «Глазами пользователя» (только для Алексея)
- Появляется кнопка в правом нижнем углу экрана
- Нажать → видишь лендинг, как его видит незарегистрированный пользователь
- Нажать снова → возврат в режим администратора

## 2. Лендинг

**Кто видит:** все, у кого нет активной подписки.

### Что показывается
- Заголовок: «Свой нутрициолог, который разберёт именно ваши анализы»
- Описание «Ваше личное пространство»
- 5 карточек функций: переписка, анализы, рекомендации, библиотека, Zoom-разборы
- Блок тарифов с переключателем «Я пенсионер(ка)»
- 3 шага «Как это работает»
- Цитата

### Тарифы на лендинге
| Тариф | Стандарт | Пенсионер (−50%) |
|-------|---------|-----------------|
| 30 дней | 3 000 ₽ | 1 500 ₽ |
| 90 дней | 7 200 ₽ | 3 600 ₽ |

**Переключатель «Я пенсионер(ка)»:**
- Цены меняются мгновенно на лендинге
- При покупке будет выбрана нужная продуктовая ссылка Prodamus
- Статус `is_senior` сохраняется в `user_access.is_senior`

### Кнопки оплаты
Ведут на Prodamus. После оплаты — пользователь попадает обратно на `/private/?payment=success`.
- Приложение 10 секунд проверяет, активировался ли доступ
- При успехе — показывается приветственный экран
- Если за 3 минуты доступ не появился — кнопка «Доступ не появился? Написать Алексею»

### Кнопка «Написать Алексею» (внизу лендинга)
Открывает модальный лист с тремя вариантами:
1. **Telegram** → @listosha
2. **MAX** → профиль в MAX
3. **Email** → listosha@list.ru

## 4. Таб «Алексей»

### Что это
Личная переписка пользователя с Алексеем. Видны только их сообщения — другие участники не видят.

### Как читать чат
- **Мои сообщения** — справа, тёмный фон
- **Сообщения Алексея** — слева, светлый фон, аватар «А»
- **Дата** — разделитель между блоками сообщений разных дней
- **Голосовое** — плеер с кнопкой ▶/⏸, прогресс-бар, время

### Как отправить сообщение
1. Напечатать текст в поле внизу
2. Нажать **Ctrl+Enter** или кнопку **➤**
3. Сообщение появится справа

### Как записать голосовое
1. Нажать **🎙** (кнопка рядом с полем ввода)
2. Появляется красный индикатор + таймер «0:00 Идёт запись...»
3. Говорить в микрофон
4. **× Отменить** — сбросить запись
5. **✓ Отправить** — загрузить и отправить
6. Максимум: **5 минут** (300 сек) — запись остановится автоматически

**Требования:** браузер должен разрешить доступ к микрофону (появится запрос).

### Типы сообщений (технически)
| thread_type | Когда создаётся |
|-------------|-----------------|
| `general` | Обычное сообщение |
| `analysis_review` | «Задать вопрос по анализу» (привязано к analysis_id) |
| `lesson_question` | Вопрос по уроку курса |

## 5. Таб «Анализы»

### Список анализов
- Карточки: иконка типа, название, лаборатория, дата
- **Бейджи:** 🔵 «новый» (не просмотрен Алексеем), 🟢 «просмотрен»
- Клик на карточку → просмотр анализа

### Загрузка анализа (4 шага)

**Шаг 1: Тип анализа**
Выбрать одну из 7 карточек:
- 🩸 Общий крови
- 🧪 Биохимия
- 🦋 Щитовидная железа
- ⚙️ Железо-панель
- 💊 Витамины
- 🔬 Гормоны
- 📋 Другое

**Шаг 2: Файл и данные**
- **Файл** (необязательно): сфотографировать или выбрать PDF, до 10 МБ
- **Лаборатория**: текстовое поле (Инвитро, Гемотест и др.)
- **Дата сдачи**: datepicker (можно пропустить)

**Шаг 3: Показатели**
- Поле поиска по маркерам — работает от 2 символов
- Выбрать маркер из выпадающего списка (77 в справочнике)
- Указать **значение** и **единицы** (dropdown)
- Кнопка «+ Добавить свой показатель» — если маркера нет в справочнике

**Шаг 4: Жалобы**
- Текстовое поле «Что беспокоит?»
- Кнопка «🎙 Записать голосовое» — голосовое пояснение

После нажатия **«✓ Сохранить»** — загружается файл (если выбран), создаётся анализ, сохраняются маркеры.

### Просмотр анализа
- Шапка: тип, лаборатория, дата
- Файл (если загружен) — кнопка «Открыть»
- Заметки (текст жалоб)
- Таблица показателей: Показатель | Значение | Ед. | (норма хранится в БД, но пока не отображается)
- Кнопка **«💬 Задать вопрос Алексею по этому анализу»** → создаёт тред типа `analysis_review`, переключает на таб «Алексей»

1. Переключить вкладку нужного раздела
2. Изменить данные
3. Нажать **«Сохранить»** — сохраняется только текущая вкладка
4. Повторить для каждой вкладки

**Важно:** каждая вкладка сохраняется отдельно. Если поменял диагнозы и препараты — нажать «Сохранить» два раза.

### Мед. карта для близкого
- В профиле добавить близкого
- После добавления — переключить профиль на вверху хаба
- Мед. карта будет сохраняться отдельно для каждого

- Включить → в модальном листе тарифы обновятся (−50%)

4 тоггла (хранятся в localStorage, не влияют на серверную логику пока):
- Ответы от Алексея
- Новые материалы
- Напоминания о разборах
- Контрольные анализы

- **«Я оплатил(а), но доступ не появился»** → создаёт тред с текстом «Проблема с доступом», открывает модалку контактов

## 9. Дневник самочувствия

### Как открыть
- Кнопка **«+ Добавить запись»** в блоке «Дневник» на хабе
- Открывается bottom sheet снизу

### Что в bottom sheet
**Верхняя часть — новая запись:**
- Текстовое поле «Как самочувствие? Что беспокоит?»
- Кнопка **«Сохранить»** → запись добавляется

**Нижняя часть — история:**
- Прокручиваемый список прошлых записей
- Каждая запись: дата + текст
- Голосовые записи: плеер

### Требования
- Текст или голосовое обязательно (одно из двух)
- Максимальный размер голосового: 5 МБ (~5 минут)

## 10. Zoom-разборы

### Предстоящие разборы
- Карточка: дата, день недели, время, название темы
- Кнопка **«Буду»** → регистрация на разбор
- После регистрации кнопка меняется на «Записан» (зелёный)
- Повторный клик: `already=true` — не создаёт дубль

### Мои записи разборов
- Видны только разборы, на которых участник был зарегистрирован
- После того, как Алексей прикрепил запись (`attach_recording`) — появляется плеер/ссылка

### Срок хранения записей
- 30 дней с даты `expires_at` в поле материала
- Автоматически скрываются pg_cron задачей ежедневно в 10:00 UTC

## 12. Онбординг нового участника

### Сценарий при первой оплате

**Шаг 1: Ожидание активации**
- Пользователь попадает на `/private/?payment=success`
- Показывается экран «🎉 Оплата получена! Активируем доступ...»
- Каждые 10 секунд проверяется `check_access`
- При активации → экран приветствия

**Шаг 2: Приветственный экран**
- «Добро пожаловать! Доступ активен до [дата]»
- Кнопка **«Заполнить карту здоровья →»** → онбординг
- Кнопка «Пропустить» → сразу на хаб

**Шаг 3: Онбординг — мед. карта (3 шага)**

*Шаг 1/3: Основное*
- Имя (предзаполнено из Telegram/MAX)
- Возраст
- Рост (см), Вес (кг)

*Шаг 2/3: Что беспокоит*
- 8 готовых диагнозов-чипов: АИТ, Анемия, Гипотиреоз, ЖКТ, Диабет 2 типа, Надпочечники, Гипертония, Остеопороз
- «+ свой» → добавить вручную
- Препараты: поле «Название» + «Доза» + кнопка «+ Добавить»
- Текстовое поле «Что беспокоит прямо сейчас?»

*Шаг 3/3: Анализы*
- Кнопка «📎 Выбрать файл» → загрузка анализа
- Кнопка «Пропустить» → на хаб

**После заполнения:**
- Данные сохраняются в `user_health_profile` и `user_analyses`
- Алексей получает уведомление в Telegram:
  ```
  📋 [Имя] заполнил(а) мед. карту
  Диагнозы: [список]
  Беспокоит: [текст]
  ```
- Флаг `private_onboarding_completed` в localStorage → повторно не показывается

## 13. Админка

**URL:** `app.listoshenkov.ru/private/admin.html`  
**Доступ:** только Алексей (user_id=1, role=admin)

### Как открыть
1. Войти в `/private/` как Алексей
2. В сайдбаре (десктоп) появляется ссылка «⚙️ Админка»
3. Или напрямую по URL

### Дашборд
4 счётчика обновляются при открытии:
| Счётчик | Что считает |
|---------|------------|
| Активных подписчиков | `has_inner_circle=true AND expires > now()` |
| Непрочитанных вопросов | Сообщения от участников, не прочитанные Алексеем |
| Новых анализов | Анализы со статусом «uploaded» |
| Истекает ≤7 дней | Подписки с expires через ≤7 дней |

Быстрые кнопки: «Переписка», «Новые анализы», «Подписчики»

### Переписка
- **Фильтры:** «Непрочитанные» / «Все»
- Список веток слева (имя участника, превью, бейдж)
- Выбранная переписка справа — хронологически
- Ответить: поле внизу + **Ctrl+Enter** или кнопка «Отправить»
- При открытии ветки — все сообщения отмечаются прочитанными (`mark_read`)
- **Бейдж «NEW»** исчезает после открытия

### Анализы
- Таблица: тип, имя участника, лаборатория, дата, статус
- **Фильтры:** «Новые» / «Просмотренные» / «Все»
- Кнопка **«✓ Просмотрено»** → статус меняется на «reviewed»
- Кнопка **«Ответить»** → открывает окно с текстом, создаёт тред `analysis_review`
- Кнопка **«+ Рекомендация»** → диалог, добавляет рекомендацию участнику

### Подписчики
Таблица со всеми активными подписчиками:
| Колонка | Описание |
|---------|----------|
| Имя | first_name из users |
| Тариф | Активен/Нет |
| Доступ до | Дата + цветной бейдж дней |
| Последний визит | last_active_at |
| Пенс. | Пометка если is_senior |

**Действия:**
- **«Продлить»** → модальное окно «Кол-во дней» → подтвердить. Продление добавляется к текущей дате истечения (не от сегодня!). Участник получает TG-уведомление.
- **«Переписка»** → переключает на таб «Переписка» с этим участником

### Контент
- Список материалов библиотеки
- Кнопка **«+ Добавить»** → форма:
  - Название (обязательно)
  - Описание
  - Тип: Статья / Аудио / Видео / PDF
  - URL (ссылка на файл или видео)
  - Теги (через запятую: `thyroid, article, understand`)
  - expires_at (необязательно) — после этой даты материал скрывается
- Кнопка **«Скрыть»** → `is_published=false`, материал исчезает из библиотеки

### Что умеет API, но нет в UI (только через прямые вызовы)
- `create_zoom_event` — создать разбор
- `attach_recording` — прикрепить запись к разбору
- `grant_recording_access` — дать доступ к записи вручную
- `add_protocol` / `edit_protocol` — протоколы для участников
- `add_marker` / `merge_custom_marker` — управление справочником маркеров
- `extend_access` — продление доступа (есть в UI)

## 15. Тарифы и оплата

### Продукты в БД
| ID | Slug | Название | Цена |
|----|------|---------|------|
| 14 | private_30 | 30 дней | 3 000 ₽ |
| 15 | private_30_senior | 30 дней (пенсионер) | 1 500 ₽ |
| 16 | private_90 | 90 дней | 7 200 ₽ |
| 17 | private_90_senior | 90 дней (пенсионер) | 3 600 ₽ |

### Что происходит при оплате
1. Пользователь нажимает «Оформить» → открывается Prodamus в браузере
2. Оплата → Prodamus отправляет webhook на `/functions/v1/prodamus-webhook`
3. Webhook ищет пользователя по telegram_id или email
4. Upsert в `user_access`:
   - `has_inner_circle = true`
   - `inner_circle_expires` = текущий expires + N дней (или от сегодня, если уже истёк)
   - `is_senior = true` если slug содержит `_senior`
5. Алексей получает уведомление

### Продление (важный нюанс)
**Добавляется к текущей дате истечения, а не от сегодня.**  
Пример: осталось 5 дней + купил 30 → будет 35 дней, не 30.

## 17. Частые вопросы

### «Оплатил, но доступ не появился»
1. Подождать 2–3 минуты (webhook обрабатывается)
2. Обновить страницу
3. Если не появился — нажать «Написать Алексею» на лендинге или в профиле → Алексей проверит вручную в Prodamus и при необходимости продлит через кнопку «Продлить» в Подписчиках

### «Не могу записать голосовое»
- Браузер должен разрешить микрофон (всплывает запрос)
- Работает только в HTTPS (на app.listoshenkov.ru — ок)
- Не работает в некоторых старых браузерах

### «Не вижу свои анализы»
- Проверить: в табе «Анализы» нет фильтрации по умолчанию — показываются все
- Если создавали анализ за «близкого» — нужно переключить профиль вверху хаба

### «Комментарии к материалу видят все?»
- Да, комментарии к материалам — публичные для всех участников Ближнего круга
- Личная переписка в табе «Алексей» — только между участником и Алексеем

### «Как Алексей добавляет новые материалы?»
1. Открыть `app.listoshenkov.ru/private/admin.html`
2. Перейти на вкладку «Контент»
3. Нажать «+ Добавить»
4. Заполнить: название, описание, тип, URL файла/видео, теги
5. Нажать «Добавить» → материал мгновенно появляется в библиотеке

### «Как продлить доступ участнику вручную?»
1. Открыть admin.html → вкладку «Подписчики»
2. Найти участника в таблице
3. Нажать «Продлить»
4. Ввести количество дней (1–365)
5. Подтвердить → участник получит TG-уведомление

### «Как создать Zoom-разбор?»
Через прямой вызов API (пока нет UI в админке):
```
POST /functions/v1/private-admin
{
  "action": "create_zoom_event",
  "token": "[admin_token]",
  "title": "Разбор анализов",
  "scheduled_at": "2026-06-01T16:00:00Z",
  "zoom_link": "https://zoom.us/j/..."
}
```
После разбора — прикрепить запись через `attach_recording`.

### «Что значит «близкий»?»
- Участник может добавить членов семьи (муж, мама, ребёнок...)
- Для каждого хранятся отдельно: мед. карта, анализы, дневник
- Переключение через аватары вверху хаба
