### T-GAME-UI-5: 🔴 КРИТИЧНО — Хаб игр тормозит ВЕЗДЕ (и телефон, и Chrome десктоп)
**Файл:** miniapp-dev/games/index.html
**Проблема:** Страница грузится минутами — и на телефоне, и в Chrome на десктопе. Не только мобильная проблема.
**Вероятная причина:** fetch к API (leaderboard или другой вызов) блокирует рендер всей страницы. Пока fetch не вернётся (или Edge Function не поднимется с холодного старта) — пользователь видит пустоту.

**Решение — страница должна отрисоваться МГНОВЕННО, без ожидания API:**

1. **Весь HTML/CSS контент (карточки игр, кнопки, заголовки) рендерится сразу.** Никаких `await fetch(...)` перед показом страницы. Контент статический — он не зависит от API.

2. **Если есть fetch к leaderboard или другому API — вынести в фон:**
```js
// НЕПРАВИЛЬНО — блокирует рендер
const data = await fetch('/functions/v1/game-action', ...);
renderPage(data);

// ПРАВИЛЬНО — страница уже на экране, данные подгружаются
document.addEventListener('DOMContentLoaded', () => {
  renderStaticContent(); // мгновенно
  loadLeaderboardAsync(); // в фоне, не блокирует
});
```

3. **Google Fonts:** `media="print" onload="this.media='all'"` (как уже сделано в iron/index.html)

4. **fetchWithTimeout()** на ВСЕ fetch-вызовы (10 сек максимум)

5. **Проверить:** `wc -c games/index.html` — если файл > 50KB, что-то не так.

6. **Проверить в DevTools:** открыть games/index.html → Network → что именно блокирует? Записать в отчёт.

**Критерий приёмки:** страница полностью отрисовывается за < 2 секунды на любом устройстве.
